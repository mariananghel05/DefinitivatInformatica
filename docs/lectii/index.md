# Lecții pe clase — cum se predă în practică (V–XII)

> Acest modul „traduce" programa în **practica reală din școlile din România**: pentru fiecare clasă —
> ce instrumente se folosesc efectiv, cum se așază conținuturile pe **modulele** anului școlar și care
> este **seria de lecții** tipică. Fiecare lecție dezvoltată are pagina ei, cu trei părți:
> **Lecția (teorie)** · **Exerciții (cu metodica aplicabilă)** · **Particularități**.

::: info Cadrul practic
- Anul școlar este organizat în **5 module de cursuri** (intervale de învățare), separate de vacanțe —
  planificarea calendaristică se face **pe module**, nu pe semestre.
- La gimnaziu, *Informatică și TIC* are **1 oră/săptămână** (OMEN 3393/2017); la liceu, numărul de ore
  depinde de profil/specializare (matematică-informatică, respectiv intensiv informatică).
- Așezarea lecțiilor pe module de mai jos este **orientativă** — reflectă practica frecventă (și ordinea
  manualelor), dar planificarea rămâne la decizia profesorului.
:::

## Instrumentele folosite în practică, pe clase

| Clasa | Algoritmi / programare | TIC / aplicații |
|---|---|---|
| **V** | **Scratch** (mediu grafic interactiv) | Paint / editoare grafice, navigare web |
| **VI** | **Scratch** (repetitive) + schemă logică, pseudocod | PowerPoint, aplicație de animație, e-mail |
| **VII** | **trecerea la C++** (Code::Blocks sau IDE online) | Word, editor audio-video, aplicații colaborative |
| **VIII** | **C++** — șiruri de valori (fără tablouri!) | Excel, editor de pagini web (HTML) |
| **IX** | pseudocod + implementare **C++** | (TIC — disciplină separată) |
| **X** | **C++**: tablouri, fișiere text, algoritmi fundamentali | — |
| **XI** | **C++**: șiruri de caractere, struct, liste, grafuri, metode de programare | — |
| **XII** | **SQL** + SGBD (baze de date); module opționale | — |

::: tip De ce Scratch → C++?
Programa de gimnaziu cere „medii grafice interactive" la V–VI și „un limbaj de programare" de la a VII-a,
fără a impune unul anume. În practică, **Scratch** domină la V–VI (feedback vizual imediat, fără erori de
sintaxă), iar de la a VII-a majoritatea școlilor trec la **C++**, pentru continuitate cu liceul,
bacalaureatul și concursurile. Unele școli aleg Python — opțiune legitimă, dar mai rară.
:::

## Cum sunt construite paginile de lecție

Fiecare lecție dezvoltată respectă același șablon, gândit pentru pregătirea de definitivat:

1. **Lecția (teorie)** — conținutul științific la nivelul clasei, cu exemplele „care funcționează" la
   vârsta respectivă;
2. **Exerciții (cu metodica aplicabilă)** — sarcini de lucru gradate, fiecare cu **metoda didactică**
   potrivită și de ce (legate de [Strategii didactice](/metodica/02-strategii-didactice));
3. **Particularități** — specificul vârstei, greșelile tipice ale elevilor, capcane de predare,
   sugestii de evaluare (legate de [Evaluarea randamentului](/metodica/03-evaluare-randament)).

> Conținutul științific „de examen" al fiecărei teme rămâne în [Tematica științifică](/stiintific/);
> paginile de lecție arată **cum ajunge el la elevi**, clasă cu clasă.
